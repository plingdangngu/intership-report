### Objectives for Week 4

* Learn about the Amazon S3 storage service.
* Understand the Object Storage model on AWS.
* Practice creating and managing Amazon S3 buckets.
* Become familiar with data management and access control features in Amazon S3.

### Tasks Completed During the Week

| Day | Tasks | Start Date | Completion Date | Reference |
| --- | ----- | ---------- | --------------- | --------- |
| Monday | Learn the fundamentals of Amazon S3, the Object Storage model, and key components such as Buckets, Objects, Keys, and Regions. | 29/05/2026 | 29/05/2026 | https://docs.aws.amazon.com/s3/ |
| Tuesday | Practice creating Amazon S3 buckets, selecting the appropriate Region, and configuring the initial bucket settings. | 30/05/2026 | 30/05/2026 | https://docs.aws.amazon.com/s3/ |
| Wednesday | Practice uploading, downloading, organizing folders, and managing data stored in Amazon S3. | 31/05/2026 | 01/06/2026 | https://docs.aws.amazon.com/s3/ |
| Thursday | Learn about Bucket Policies, IAM Permissions, Versioning, and Lifecycle Rules for secure data management. | 02/06/2026 | 03/06/2026 | https://docs.aws.amazon.com/s3/ |
| Friday | Review the concepts learned, verify bucket configurations, and practice managing data in Amazon S3. | 04/06/2026 | 05/06/2026 | https://docs.aws.amazon.com/s3/ |

### Results Achieved in Week 4

* Gained an understanding of how Amazon S3 works and the Object Storage model on AWS.

* Successfully created and configured Amazon S3 buckets for data storage.

* Performed common operations such as uploading, downloading, updating, and deleting objects in Amazon S3.

* Learned how to manage access permissions using IAM and Bucket Policies.

* Understood how Versioning can be used to maintain multiple versions of objects and reduce the risk of accidental data loss.

* Learned how Lifecycle Rules can automatically manage data throughout different storage phases.

* Acquired the skills to use Amazon S3 as a storage solution for applications deployed on AWS.
