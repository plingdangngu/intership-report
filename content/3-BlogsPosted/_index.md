### [Blog 1 - Building an AI Gateway for Amazon Bedrock with Amazon API Gateway](3.1-Blog1/)

This blog introduces the AI Gateway architecture on AWS, which uses Amazon API Gateway as an intermediary layer between applications and Amazon Bedrock to centrally manage access to foundation models. It explains the roles of core AWS services, including Amazon API Gateway, AWS Lambda, IAM, and Amazon Bedrock, while analyzing the request flow, security benefits, scalability, cost control, and the deployment process using AWS CloudFormation.

### [Blog 2 - Deploying an AI Gateway for Amazon Bedrock on AWS](3.2-Blog2/)

This blog provides a step-by-step guide to deploying an AI Gateway on AWS using AWS CloudFormation. It covers environment preparation, infrastructure deployment, gateway validation, Response Streaming, and potential enhancements such as AWS WAF, Amazon CloudWatch, API Versioning, and Canary Deployment to improve security, monitoring, and scalability.

### [Blog 3 - Hosting a .NET Blazor WebAssembly Application on Amazon S3 and Amazon CloudFront](3.3-Blog3/)

This blog demonstrates how to deploy a .NET Blazor WebAssembly application using a serverless architecture with Amazon S3 and Amazon CloudFront. It explains the roles of Blazor WebAssembly, Amazon S3, Amazon CloudFront, and Terraform, and provides guidance on publishing the application, provisioning infrastructure, uploading the website to Amazon S3, and improving website performance through CloudFront's global content delivery network.
