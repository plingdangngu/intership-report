#### IAM role separation

SmartStudy uses AWS IAM roles to separate infrastructure-deployment permissions from application runtime permissions. AWS CDK creates bootstrap roles for CloudFormation execution, deployment, asset publishing, and context lookup. The SmartStudy stacks create additional execution roles for the Lambda functions in each environment.

The role inventory confirms the presence of:

* CDK CloudFormation execution and deployment roles.
* CDK file-publishing, image-publishing, and lookup roles.
* API Lambda execution roles for staging and production.
* Document Ingestion Lambda execution roles for staging and production.
* Pre Sign-up Lambda execution roles for staging and production.

The Lambda service is listed as the trusted entity for the SmartStudy execution roles, while AWS CloudFormation is trusted by the CDK CloudFormation execution role.

#### Customer-managed policies

The IAM policy inventory includes customer-managed policies associated with the SmartStudy API Lambda roles in staging and production. It also contains a Lambda basic-execution policy used to support runtime logging. Some policies generated by CDK use an overflow-policy name when the permissions cannot remain entirely within the primary inline policy.

![Customer-managed IAM policies](/intership-report/images/5-Workshop/5.3-Infrastructure/iam-policies.png)

*Figure 5.17: Customer-managed policies, including policies generated for SmartStudy API functions.*

These screenshots verify the role and policy separation at a high level. They do not expose or claim every individual IAM action. Detailed permissions remain defined by the CDK source and should follow the principle of least privilege: each function receives only the access required for its own workflow.

#### Security result

The IAM structure provides:

* Separate deployment and runtime trust boundaries.
* Environment-specific Lambda execution roles.
* Service-based trust relationships rather than long-lived application credentials.
* Repeatable role and policy creation through AWS CDK and CloudFormation.
