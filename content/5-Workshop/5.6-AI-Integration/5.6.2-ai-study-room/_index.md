#### Document-grounded conversation

The SmartStudy study room combines the conversation interface with the active source document. A learner selects a ready document, submits a question, and receives a response generated from the document context.

The interface keeps the document visible beside the conversation so that the learner can compare the AI response with the source material.

![SmartStudy AI study room](/intership-report/images/5-Workshop/5.6-AI-Integration/ai-study-room.png)

*Figure 5.30: Document conversation and source document displayed in the same study workspace.*

#### Learning assistant

The learning-assistant mode supports focused requests such as explaining a concept, providing an example, or summarizing a selected section. The option to base the response on the open document keeps the interaction aligned with the learner's source.

![SmartStudy learning assistant](/intership-report/images/5-Workshop/5.6-AI-Integration/ai-assistant.png)

*Figure 5.31: Learning-assistant response generated from the active document.*

These interfaces verify that the deployed application can submit AI requests and present generated responses within the document-learning workflow. Infrastructure secrets and provider endpoints remain hidden from the browser-facing report.
